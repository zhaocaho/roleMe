# 用户记忆

角色名：**`<role-name>`**

## 用途

存储稳定的记忆偏好、长期约定和持久的角色特定记忆。

**核心原则**：这里存放的内容**忘记了也不会改变角色的身份**。

## 分类

### 长期偏好

应该跨对话持久化的偏好：

```markdown
长期偏好：
- 语言偏好：[中文 / 英文 / ...]
- 回答风格：[结论先行 / 详细展开 / ...]
- 技术倾向：[具体偏好]
- 工作时间：[相关信息，如有]
```

### 稳定约定

与用户达成的应该被记住的约定：

```markdown
已确认约定：
- [约定1]: [具体内容]
- [约定2]: [具体内容]
```

### 持久事实

会反复涉及的用户或上下文事实：

```markdown
持久事实：
- 用户背景：[相关信息]
- 项目状态：[当前主要项目]
- 技术栈偏好：[主要使用的技术]
```

## 示例内容

```markdown
长期偏好：
- 默认中文沟通
- 回答先给结论，再补充细节
- 代码偏好可读性优于简洁性

已确认约定：
- 每次提交代码前先自查变更范围
- 重要决策需给出至少两个选项

持久事实：
- 用户是前端开发工程师
- 当前主要项目：roleMe 角色系统
```

## 指导原则

- 只存储**稳定、持久的信息**
- 会话特定的笔记不属于这里
- 如果删除内容只意味着角色"忘记了一些事"，它属于这里
- 如果删除内容会改变角色是谁，应该移到 identity.md
