# 记忆索引

角色名：**`<role-name>`**

## 用途

这个文件是**记忆摘要和记忆组织的入口点**。

它应该：
- 指向稳定的记忆主题
- 摘要记录已记住的内容
- 帮助定位具体的记忆内容

## 记忆结构

### 核心记忆文件

| 文件 | 用途 |
|------|------|
| `USER.md` | 用户偏好、约定和持久事实 |

### 记忆主题

如果角色有额外的记忆文件，在这里列出：

```markdown
记忆主题：
- [主题1]: [文件路径或简要描述]
- [主题2]: [文件路径或简要描述]
```

## 当前记忆概要

简要概括角色当前记住的内容：

```markdown
当前记忆概要：
- [记忆类别1]: [关键内容摘要]
- [记忆类别2]: [关键内容摘要]
```

## 示例结构

```markdown
记忆主题：
- 项目记忆: projects/<项目名>/memory.md (如有)
- 技术偏好: brain/tech-preferences.md (如有)

当前记忆概要：
- 用户偏好: 默认中文，结论先行
- 工作约定: 决策需多选项，变更需自查
- 持久事实: 前端工程师，当前项目 roleMe
```

## 指导原则

- 这个文件是**索引**，不是内容堆砌
- 随着记忆增长保持更新
- 有具体记忆文件时链接到它们
- 移除过时条目以保持清晰
