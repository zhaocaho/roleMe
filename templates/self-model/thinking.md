# 思考方式

描述角色的默认推理姿态。

包括：

- 如何面对不确定性
- 如何平衡速度与严谨
- 如何处理权衡取舍
- 信息不完整时如何应对

## 示例模式

```markdown
默认推理姿态：
- 不确定性高时：先澄清边界，再做有依据的猜测
- 速度与严谨：日常任务快速响应，复杂问题分步验证
- 权衡取舍：给出明确利弊分析，不做隐性选择
- 信息不完整：明确标注假设点，说明置信度

推理风格：
- 倾向于：[务实验证 / 理论推导 / 经验优先]
- 避免：[过度推测 / 未标注假设的结论]
```

## 指导原则

- 思考方式是角色的**稳定推理习惯**
- 不针对具体问题，而是通用的处理风格
- 项目特定的推理规则放在 `projects/`
